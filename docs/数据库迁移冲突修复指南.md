# 数据库迁移冲突修复指南

## 📋 概述

本文档记录了在数据库迁移历史混乱时，如何通过创建基线迁移和手动升级数据库来解决问题的完整流程。

## 🎯 适用场景

当遇到以下情况时，可以使用本指南：
- 旧的迁移文件丢失或损坏
- 数据库结构与当前 `schema.zmodel` 不匹配
- 迁移历史中出现失败的迁移记录
- 需要在生产环境中同步数据库结构而不丢失数据

## ⚠️ 重要前提条件

1. **数据备份**：操作前必须备份数据库
2. **风险意识**：涉及生产数据库操作，需谨慎
3. **测试验证**：建议先在测试环境验证流程

## 🔧 完整操作流程

### 第一步：分析现状

#### 1.1 检查本地迁移状态

```bash
# 查看本地迁移文件
ls -la migrations/

# 查看当前 schema
cat schema.zmodel
```

#### 1.2 检查服务器数据库结构

```bash
# 使用导出脚本查看服务器数据库结构
ssh root@your-server 'cd /path/to/project && node export-db-schema.js prisma/dev.db'

# 或者查看特定表结构
ssh root@your-server 'cd /path/to/project && node export-db-schema.js prisma/dev.db | grep "CREATE TABLE"'
```

#### 1.3 对比差异

重点检查：
- 表结构差异（新增/删除的字段）
- 索引差异
- 外键约束差异
- 数据类型变更

### 第二步：创建服务器数据库备份

```bash
# SSH 到服务器
ssh root@your-server

# 备份数据库（重要！）
cd /path/to/project
cp prisma/dev.db prisma/dev.db.backup_$(date +%Y%m%d_%H%M%S)

# 验证备份
ls -lh prisma/dev.db.backup_*
```

### 第三步：在本地创建基线迁移

```bash
# 在项目根目录执行
pnpm zenstack migrate dev --name baseline

# 这会生成类似这样的文件：
# migrations/20260303015857_baseline/migration.sql
```

### 第四步：分析并生成升级脚本

#### 4.1 读取生成的迁移文件

```bash
cat migrations/20260303015857_baseline/migration.sql
```

#### 4.2 对比服务器和本地结构差异

使用 `export-db-schema.js` 导出两边结构进行对比：

```bash
# 本地数据库结构（如果存在）
node export-db-schema.js apps/backend/database/dev.db > local_schema.txt

# 服务器数据库结构
ssh root@your-server 'cd /path/to/project && node export-db-schema.js prisma/dev.db' > server_schema.txt

# 对比差异
diff local_schema.txt server_schema.txt
```

#### 4.3 创建升级脚本

根据差异分析，创建针对性的 SQL 升级脚本。

**示例：Post 表结构升级**

```sql
-- ========================================
-- 数据库升级脚本
-- 风险：⚠️ 涉及表结构变更，请先备份！
-- ========================================

-- 第一步：备份现有数据（如果有的话）
CREATE TABLE IF NOT EXISTS "Post_backup" AS SELECT * FROM "Post";

-- 第二步：删除旧表
DROP TABLE IF EXISTS "Post";

-- 第三步：创建新表（匹配 schema.zmodel）
CREATE TABLE "Post" (
    "created" DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    "updated" DATETIME NOT NULL,
    "id" INTEGER NOT NULL PRIMARY KEY AUTOINCREMENT,
    "title" TEXT,
    "content" TEXT NOT NULL,
    "visibility" TEXT NOT NULL DEFAULT 'DRAFT',
    "authorId" TEXT NOT NULL,
    "parentId" INTEGER,
    CONSTRAINT "Post_authorId_fkey" FOREIGN KEY ("authorId") REFERENCES "User" ("id") ON DELETE RESTRICT ON UPDATE CASCADE,
    CONSTRAINT "Post_parentId_fkey" FOREIGN KEY ("parentId") REFERENCES "Post" ("id") ON DELETE SET NULL ON UPDATE CASCADE
);

-- 第四步：创建索引
CREATE INDEX "Post_parentId_created_idx" ON "Post"("parentId", "created");
CREATE INDEX "Post_visibility_created_idx" ON "Post"("visibility", "created");

-- 第五步：数据迁移（如果需要）
-- 根据实际情况编写数据迁移逻辑
```

### 第五步：执行服务器数据库升级

#### 5.1 创建升级执行脚本

创建 `execute-upgrade.js`：

```javascript
#!/usr/bin/env node
import Database from 'better-sqlite3';
import fs from 'fs';

function main() {
  const dbPath = process.argv[2] || './prisma/dev.db';

  console.log('🔧 数据库升级工具');

  const db = new Database(dbPath);
  db.pragma('foreign_keys = OFF'); // 临时关闭外键约束

  // 执行升级 SQL
  const sql = fs.readFileSync('./upgrade.sql', 'utf-8');

  // 分步执行 SQL 语句
  const statements = sql
    .split(';')
    .map(s => s.trim())
    .filter(s => s.length > 0 && !s.startsWith('--'));

  for (const statement of statements) {
    try {
      db.exec(statement);
      console.log('✅ 执行成功:', statement.substring(0, 50) + '...');
    } catch (error) {
      console.error('❌ 执行失败:', error.message);
    }
  }

  db.pragma('foreign_keys = ON');
  db.close();
  console.log('✅ 升级完成');
}

main();
```

#### 5.2 在服务器上执行升级

```bash
# 上传脚本到服务器
scp execute-upgrade.js upgrade.sql root@your-server:/path/to/project/

# SSH 到服务器执行
ssh root@your-server 'cd /path/to/project && node execute-upgrade.js prisma/dev.db'
```

### 第六步：清理并重置迁移历史

#### 6.1 创建清理脚本

创建 `clear-migrations.js`：

```javascript
#!/usr/bin/env node
import Database from 'better-sqlite3';
import fs from 'fs';

function main() {
  const dbPath = process.argv[2] || './prisma/dev.db';

  console.log('🧹 迁移历史清理工具');

  const db = new Database(dbPath);

  // 第一步：备份当前迁移记录
  const oldMigrations = db.prepare("SELECT * FROM _prisma_migrations").all();
  console.log(`找到 ${oldMigrations.length} 条旧迁移记录`);

  if (oldMigrations.length > 0) {
    const backupFile = `migrations_backup_${Date.now()}.json`;
    fs.writeFileSync(backupFile, JSON.stringify(oldMigrations, null, 2));
    console.log(`✅ 已备份到: ${backupFile}`);
  }

  // 第二步：删除所有旧迁移记录
  db.prepare("DELETE FROM _prisma_migrations").run();
  console.log('✅ 已删除所有旧迁移记录');

  // 第三步：添加基线迁移记录
  const migrationName = '20260303015857_baseline'; // 替换为你的迁移名
  const checksum = Math.random().toString(36).substring(2, 15);
  const now = new Date().toISOString().replace('T', ' ').replace(/\.\d{3}Z$/, '');

  db.prepare(`
    INSERT INTO _prisma_migrations (id, checksum, finished_at, migration_name, started_at, applied_steps_count)
    VALUES (?, ?, ?, ?, ?, ?)
  `).run(`${migrationName}_${Date.now()}`, checksum, now, migrationName, now, 1);

  console.log('✅ 基线迁移已添加');
  console.log(`   名称: ${migrationName}`);
  console.log(`   时间: ${now}`);

  db.close();
  console.log('✅ 清理完成');
}

main();
```

#### 6.2 在服务器上执行清理

```bash
# 上传脚本
scp clear-migrations.js root@your-server:/path/to/project/

# 执行清理
ssh root@your-server 'cd /path/to/project && node clear-migrations.js prisma/dev.db'
```

## 🔍 关键注意事项

### 数据安全

1. **必须备份**：操作前必须备份数据库
2. **分步验证**：每步操作后都要验证结果
3. **回滚预案**：准备好回滚方案

### 表结构变更

1. **数据迁移**：如果表有重要数据，需要先导出再导入
2. **字段映射**：注意字段类型的兼容性和数据映射关系
3. **外键约束**：操作前关闭外键约束，操作后重新启用

### 迁移历史

1. **记录备份**：清理前备份旧的迁移记录
2. **基线命名**：使用有意义的命名（如 `baseline_YYYYMMDD`）
3. **校验和**：迁移记录的校验和可以随机生成（基线迁移）

## 📊 常见问题排查

### 问题 1：迁移执行失败

**错误信息**：`migrate found failed migrations in the target database`

**解决方案**：
```bash
# 清理失败的迁移历史
node clear-migrations.js prisma/dev.db
```

### 问题 2：外键约束冲突

**错误信息**：`FOREIGN KEY constraint failed`

**解决方案**：
```javascript
// 在升级脚本中临时关闭外键约束
db.pragma('foreign_keys = OFF');
// 执行表结构变更...
// 完成后重新启用
db.pragma('foreign_keys = ON');
```

### 问题 3：数据丢失

**预防措施**：
1. 操作前必须备份
2. 对于重要数据表，先创建备份表
3. 分步骤执行，每步验证

**恢复方法**：
```bash
# 从备份恢复
cp prisma/dev.db.backup_YYYYMMDD_HHMMSS prisma/dev.db
```

### 1. 数据库结构导出脚本

**用途**：导出 SQLite 数据库的完整表结构、索引等信息

```javascript
// export-db-schema.js
#!/usr/bin/env node
/**
 * SQLite 数据库结构导出工具
 *
 * 用途：导出数据库的完整 schema（表结构、索引、外键等）
 * 输出：直接输出到控制台，可重定向到文件
 *
 * 使用方法：
 *   node export-db-schema.js /path/to/database.db
 *   node export-db-schema.js /path/to/database.db > schema.txt
 *
 * 依赖：
 *   npm install better-sqlite3
 */

import Database from 'better-sqlite3';
import fs from 'fs';

const dbPath = process.argv[2];

if (!dbPath) {
  console.error('❌ 错误：请提供数据库路径');
  console.log('\n使用方法:');
  console.log('  node export-db-schema.js /path/to/database.db');
  console.log('\n示例:');
  console.log('  node export-db-schema.js ./prisma/dev.db');
  console.log('  node export-db-schema.js ./prisma/dev.db > schema.txt');
  process.exit(1);
}

if (!fs.existsSync(dbPath)) {
  console.error(`❌ 错误：数据库文件不存在: ${dbPath}`);
  process.exit(1);
}

try {
  const db = new Database(dbPath, { readonly: true });

  console.log('-- SQLite Database Schema');
  console.log(`-- Generated: ${new Date().toISOString()}`);
  console.log(`-- Database: ${dbPath}\n`);

  // 导出所有对象（表、索引、触发器等）
  const schema = db.prepare("SELECT sql FROM sqlite_master WHERE sql IS NOT NULL ORDER BY type, name").all();

  schema.forEach(row => {
    console.log(row.sql + ';\n');
  });

  // 统计信息
  const tables = db.prepare("SELECT name FROM sqlite_master WHERE type='table' ORDER BY name").all();
  console.log(`\n-- 统计信息: 共 ${tables.length} 个表`);

  db.close();
} catch (error) {
  console.error(`❌ 导出失败: ${error.message}`);
  process.exit(1);
}
```

### 2. 迁移状态验证脚本

**用途**：查看数据库中已应用的迁移记录

```javascript
// verify-migrations.js
#!/usr/bin/env node
/**
 * 迁移状态验证工具
 *
 * 用途：查看数据库中已应用的迁移记录
 *
 * 使用方法：
 *   node verify-migrations.js /path/to/database.db
 *
 * 依赖：
 *   npm install better-sqlite3
 */

import Database from 'better-sqlite3';
import fs from 'fs';

const dbPath = process.argv[2] || './prisma/dev.db';

if (!fs.existsSync(dbPath)) {
  console.error(`❌ 数据库不存在: ${dbPath}`);
  process.exit(1);
}

try {
  const db = new Database(dbPath);

  // 检查迁移表是否存在
  const tableExists = db.prepare(
    "SELECT name FROM sqlite_master WHERE type='table' AND name='_prisma_migrations'"
  ).get();

  if (!tableExists) {
    console.log('❌ _prisma_migrations 表不存在');
    process.exit(1);
  }

  // 查询迁移记录
  const migrations = db.prepare(`
    SELECT migration_name, started_at, finished_at, applied_steps_count
    FROM _prisma_migrations
    ORDER BY applied_steps_count ASC
  `).all();

  console.log('━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━');
  console.log('📋 迁移记录');
  console.log('━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\n');
  console.log(`总计: ${migrations.length} 条迁移记录\n`);

  migrations.forEach((m, index) => {
    const status = m.finished_at ? '✅ 已完成' : '⏳ 进行中';
    const num = index + 1;
    console.log(`${num}. ${m.migration_name}`);
    console.log(`   状态: ${status}`);
    console.log(`   开始: ${m.started_at}`);
    if (m.finished_at) {
      console.log(`   完成: ${m.finished_at}`);
    }
    console.log(`   步骤: ${m.applied_steps_count}\n`);
  });

  db.close();
} catch (error) {
  console.error(`❌ 错误: ${error.message}`);
  process.exit(1);
}
```

### 3. 迁移历史清理脚本

**用途**：清理所有迁移记录并添加基线迁移

```javascript
// clear-migrations.js
#!/usr/bin/env node
/**
 * 迁移历史清理工具
 *
 * 用途：清理所有迁移记录并添加基线迁移记录
 *
 * 使用方法：
 *   node clear-migrations.js /path/to/database.db [baseline_migration_name]
 *
 * 示例：
 *   node clear-migrations.js ./prisma/dev.db
 *   node clear-migrations.js ./prisma/dev.db 20260303015857_baseline
 *
 * 依赖：
 *   npm install better-sqlite3
 */

import Database from 'better-sqlite3';
import fs from 'fs';

const dbPath = process.argv[2] || './prisma/dev.db';
const baselineMigration = process.argv[3] || 'baseline';

if (!fs.existsSync(dbPath)) {
  console.error(`❌ 数据库不存在: ${dbPath}`);
  process.exit(1);
}

try {
  const db = new Database(dbPath);

  console.log('━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━');
  console.log('🧹 迁移历史清理工具');
  console.log('━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\n');

  // 第一步：备份当前迁移记录
  console.log('📋 第一步：备份当前迁移记录');
  const oldMigrations = db.prepare("SELECT * FROM _prisma_migrations").all();

  if (oldMigrations.length > 0) {
    const backupFile = `migrations_backup_${Date.now()}.json`;
    fs.writeFileSync(backupFile, JSON.stringify(oldMigrations, null, 2));
    console.log(`   ✅ 已备份 ${oldMigrations.length} 条记录到: ${backupFile}\n`);
  } else {
    console.log('   ℹ️  没有找到旧迁移记录\n');
  }

  // 第二步：删除所有迁移记录
  console.log('🗑️  第二步：删除所有迁移记录');
  const deleteResult = db.prepare("DELETE FROM _prisma_migrations").run();
  console.log(`   ✅ 已删除 ${deleteResult.changes} 条记录\n`);

  // 第三步：添加基线迁移记录
  console.log('📝 第三步：添加基线迁移记录');
  const checksum = Math.random().toString(36).substring(2, 15);
  const now = new Date().toISOString().replace('T', ' ').replace(/\.\d{3}Z$/, '');

  const insert = db.prepare(`
    INSERT INTO _prisma_migrations (id, checksum, finished_at, migration_name, started_at, applied_steps_count)
    VALUES (?, ?, ?, ?, ?, ?)
  `);

  const migrationId = `${baselineMigration}_${Date.now()}`;
  insert.run(migrationId, checksum, now, baselineMigration, now, 1);

  console.log(`   ✅ 基线迁移已添加`);
  console.log(`      名称: ${baselineMigration}`);
  console.log(`      ID: ${migrationId}`);
  console.log(`      时间: ${now}\n`);

  // 第四步：验证结果
  console.log('🔍 第四步：验证结果');
  const currentMigrations = db.prepare(
    "SELECT migration_name, finished_at FROM _prisma_migrations"
  ).all();

  console.log(`   当前迁移记录数: ${currentMigrations.length}`);
  currentMigrations.forEach(m => {
    console.log(`   - ${m.migration_name} (${m.finished_at})`);
  });

  db.close();

  console.log('\n━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━');
  console.log('✅ 迁移历史清理完成！');
  console.log('━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\n');

} catch (error) {
  console.error(`❌ 错误: ${error.message}`);
  process.exit(1);
}
```

### 4. 数据库升级执行脚本

**用途**：执行 SQL 升级脚本

```javascript
// execute-upgrade.js
#!/usr/bin/env node
/**
 * 数据库升级执行工具
 *
 * 用途：读取并执行 SQL 升级脚本
 *
 * 使用方法：
 *   node execute-upgrade.js /path/to/database.db /path/to/upgrade.sql
 *
 * 依赖：
 *   npm install better-sqlite3
 */

import Database from 'better-sqlite3';
import fs from 'fs';

const dbPath = process.argv[2] || './prisma/dev.db';
const sqlFilePath = process.argv[3] || './upgrade.sql';

if (!fs.existsSync(dbPath)) {
  console.error(`❌ 数据库不存在: ${dbPath}`);
  process.exit(1);
}

if (!fs.existsSync(sqlFilePath)) {
  console.error(`❌ SQL 文件不存在: ${sqlFilePath}`);
  process.exit(1);
}

try {
  console.log('━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━');
  console.log('🔧 数据库升级工具');
  console.log('━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\n');

  console.log(`📄 数据库: ${dbPath}`);
  console.log(`📄 SQL 脚本: ${sqlFilePath}\n`);

  const db = new Database(dbPath);
  db.pragma('foreign_keys = OFF'); // 临时关闭外键约束

  const sql = fs.readFileSync(sqlFilePath, 'utf-8');

  console.log('⚙️  正在执行升级脚本...\n');

  // 分割 SQL 语句并执行
  const statements = sql
    .split(';')
    .map(s => s.trim())
    .filter(s => s.length > 0 && !s.startsWith('--'));

  let successCount = 0;
  let errorCount = 0;

  for (const statement of statements) {
    try {
      db.exec(statement);
      successCount++;
      console.log(`✅ [${successCount}] ${statement.substring(0, 60)}...`);
    } catch (error) {
      errorCount++;
      console.error(`❌ 执行失败: ${error.message}`);
      console.error(`📝 SQL: ${statement.substring(0, 100)}...\n`);
    }
  }

  db.pragma('foreign_keys = ON'); // 重新启用外键约束
  db.close();

  console.log('\n━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━');
  console.log('📊 执行结果');
  console.log('━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\n');
  console.log(`✅ 成功: ${successCount} 条语句`);
  console.log(`❌ 失败: ${errorCount} 条语句`);

  if (errorCount === 0) {
    console.log('\n✅ 数据库升级成功！');
  } else {
    console.log('\n⚠️  升级过程中遇到错误，请检查日志');
  }

} catch (error) {
  console.error(`❌ 升级失败: ${error.message}`);
  process.exit(1);
}
```

### 5. 数据库差异对比脚本

**用途**：对比两个数据库的结构差异

```bash
#!/bin/bash
# compare-dbs.sh
#
# 数据库结构对比工具
#
# 用途：对比两个数据库的结构差异
#
# 使用方法：
#   bash compare-dbs.sh /path/to/db1.db /path/to/db2.db
#   bash compare-dbs.sh local.db server.db > diff.txt
#
# 依赖：
#   node, better-sqlite3

DB1=${1:-"./local.db"}
DB2=${2:-"./server.db"}
EXPORT_SCRIPT=${3:-"./export-db-schema.js"}

echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo "🔍 数据库结构对比工具"
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo ""

# 检查导出脚本是否存在
if [ ! -f "$EXPORT_SCRIPT" ]; then
  echo "❌ 错误：找不到导出脚本 $EXPORT_SCRIPT"
  echo "请先创建 export-db-schema.js 脚本"
  exit 1
fi

# 检查数据库文件是否存在
if [ ! -f "$DB1" ]; then
  echo "❌ 错误：数据库1不存在: $DB1"
  exit 1
fi

if [ ! -f "$DB2" ]; then
  echo "❌ 错误：数据库2不存在: $DB2"
  exit 1
fi

echo "📄 数据库1: $DB1"
echo "📄 数据库2: $DB2"
echo ""

# 导出数据库结构
echo "⏳ 正在导出数据库1结构..."
node "$EXPORT_SCRIPT" "$DB1" > /tmp/db1_schema.txt 2>/dev/null
if [ $? -ne 0 ]; then
  echo "❌ 导出数据库1失败"
  exit 1
fi
echo "✅ 数据库1结构已导出"

echo "⏳ 正在导出数据库2结构..."
node "$EXPORT_SCRIPT" "$DB2" > /tmp/db2_schema.txt 2>/dev/null
if [ $? -ne 0 ]; then
  echo "❌ 导出数据库2失败"
  exit 1
fi
echo "✅ 数据库2结构已导出"

echo ""
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo "📊 差异对比结果"
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo ""

# 对比差异
diff -u /tmp/db1_schema.txt /tmp/db2_schema.txt || true

echo ""
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo "✅ 对比完成"
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo ""
echo "💡 提示："
echo "   - 以 '-' 开头的行表示只在数据库1中存在"
```
